# hydra/runtime/memory

## Responsibility

This directory manages the persistent, semantically searchable memory system. Memory in Hydra is not a chat log. It is a collection of typed knowledge nodes that represent meaningful facts, decisions, tasks, bugs, and code entities extracted from interactions with the model and the workspace.

Memory nodes are stored in SQLite and indexed in a FAISS vector index using embeddings generated by the sentence-transformers library. When the compiler needs context, it queries the memory store using the user's query and retrieves the most semantically relevant nodes.

---

## Rules for This Directory

- This directory is the only place where FAISS index operations and embedding generation should occur.
- The memory store must expose an abstract interface so that the backing implementation (SQLite plus FAISS, Qdrant, Chroma, etc.) can be swapped without changing any other module.
- Embedding generation must be lazy and cached where possible to avoid repeated model loads.
- The FAISS index must be saved to disk after every write operation so that it survives process restarts.
- Memory node schemas are defined in `hydra/models/schemas.py`, not in this directory.

---

## Files

### \_\_init\_\_.py

Marks this directory as a Python package.

### store.py

**Purpose**: Implements the `MemoryStore` class that provides the complete interface for reading and writing memory nodes.

**What this file must contain**:

- The `MemoryStore` class.
- An `add(node: MemoryNode) -> str` method that:
  1. Generates an embedding for the node's text content using sentence-transformers.
  2. Saves the node record to the SQLite database via the ORM.
  3. Adds the embedding to the FAISS index.
  4. Saves the FAISS index to disk.
  5. Returns the assigned node ID.
- A `search(query: str, top_k: int) -> list[MemoryNode]` method that:
  1. Generates an embedding for the query string.
  2. Queries the FAISS index for the top-k most similar vectors.
  3. Retrieves the corresponding node records from SQLite.
  4. Returns the nodes sorted by relevance score.
- A `get_by_id(node_id: str) -> MemoryNode` method.
- A `get_by_type(node_type: str) -> list[MemoryNode]` method.
- A `delete(node_id: str) -> None` method that removes the node from both SQLite and the FAISS index.
- A `clear(workspace_id: str) -> None` method that removes all nodes for a specific workspace.
- Private methods for loading and saving the FAISS index from disk.
- Private methods for loading the sentence-transformers embedding model (loaded once and cached).

**What this file must not contain**:
- Prompt construction.
- Router logic.
- HTTP calls.
- Any logic not related to memory storage and retrieval.

## Memory Node Types

The following node types are currently defined. New types may be added as the project evolves.

| Type         | Description                                              |
|--------------|----------------------------------------------------------|
| `decision`   | An architectural or design decision made during a session |
| `task`       | An active or completed development task                  |
| `bug`        | A recorded bug or issue                                  |
| `file`       | A reference to a specific file in the workspace          |
| `class`      | A class or type defined in the codebase                  |
| `function`   | A function or method defined in the codebase             |
| `api`        | An API endpoint or interface                             |
| `constraint` | A project-level constraint or requirement                |
